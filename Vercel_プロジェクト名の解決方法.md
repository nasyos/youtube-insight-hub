# Vercel プロジェクト名の解決方法

## ⚠️ 問題

「同じ名前は登録できません」というエラーが表示される場合、以下のいずれかが原因です：

1. **既存のプロジェクト名と重複している**
2. **既存のプロジェクトが存在している**

---

## ✅ 解決方法

### **方法1: 既存のプロジェクトを使用（推奨）**

既存のプロジェクトがある場合は、それを使用するのが最も簡単です。

#### **ステップ1: 既存プロジェクトを確認**

1. Vercelダッシュボードにアクセス
2. プロジェクト一覧を確認
3. `youtube-insight-hub` という名前のプロジェクトがあるか確認

#### **ステップ2: 既存プロジェクトを選択**

1. 既存のプロジェクトをクリック
2. **Settings** → **General**
3. プロジェクトの設定を確認

#### **ステップ3: 環境変数を設定**

1. **Settings** → **Environment Variables**
2. 必要な環境変数が設定されているか確認
3. 不足している場合は追加：
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `YOUTUBE_API_KEY`
   - `GEMINI_API_KEY`
   - `API_KEY`

#### **ステップ4: 最新のコードをデプロイ**

**GitHubリポジトリが接続されている場合**:
- 自動的にデプロイされます
- または、**Deployments**タブで「**Redeploy**」をクリック

**GitHubリポジトリが接続されていない場合**:
```powershell
vercel --prod
```

---

### **方法2: プロジェクト名を変更**

新しいプロジェクト名を使用する場合。

#### **ステップ1: プロジェクト名を変更**

Vercelダッシュボードで：
1. プロジェクト設定画面で「**Project Name**」を変更
2. 例: `youtube-insight-hub-v2` や `youtube-insight-hub-new`
3. 「**Save**」をクリック

または、新規プロジェクト作成時に：
- **Project Name**に別の名前を入力（例: `youtube-insight-hub-2025`）

---

### **方法3: GitHubリポジトリを接続（推奨・長期的）**

GitHubリポジトリを接続すると、自動デプロイが有効になります。

#### **ステップ1: GitHubリポジトリをプッシュ**

```powershell
# 現在の変更をコミット
git add .
git commit -m "APIキー認証の実装とVercel設定"

# GitHubにプッシュ
git push origin main
```

#### **ステップ2: VercelでGitHubリポジトリを接続**

1. Vercelダッシュボードで「**Add New...**」→「**Project**」
2. 「**Connect GitHub**」をクリック（まだ接続していない場合）
3. リポジトリ `youtube-insight-hub` を選択
4. 「**Import**」をクリック

#### **ステップ3: プロジェクト設定**

1. **Project Name**: 任意の名前（既存のものと重複しない名前）
2. **Framework Preset**: `Vite`
3. 環境変数を設定（方法1のステップ3を参照）

#### **ステップ4: デプロイ**

1. 「**Deploy**」をクリック
2. 自動的にGitHubからコードが取得されてデプロイされます

---

## 🎯 推奨手順

### **今すぐデプロイしたい場合**

1. ✅ **既存のプロジェクトを使用**（方法1）
   - 最も簡単
   - すぐにデプロイできる

2. ✅ **環境変数を設定**
   - 既存プロジェクトのSettings → Environment Variables

3. ✅ **デプロイ**
   - GitHub接続済み: 自動デプロイ
   - 未接続: `vercel --prod`

### **長期的に運用する場合**

1. ✅ **GitHubリポジトリを接続**（方法3）
   - 自動デプロイが有効になる
   - コードをプッシュするだけでデプロイされる

---

## 📝 既存プロジェクトの確認方法

### **Vercelダッシュボードで確認**

1. [Vercelダッシュボード](https://vercel.com/dashboard)にアクセス
2. プロジェクト一覧を確認
3. `youtube-insight-hub` という名前のプロジェクトがあるか確認

### **CLIで確認**

```powershell
# プロジェクト一覧を確認
vercel ls
```

---

## 🔄 既存プロジェクトにリンクする方法

### **CLIからリンク**

```powershell
vercel link
```

以下の質問に答えます：

1. **Set up and develop?** → `Y`
2. **Which scope?** → アカウントを選択
3. **Link to existing project?** → `Y`（既存プロジェクトを使用）
4. **What's your project's name?** → 既存のプロジェクト名を選択

---

## ⚠️ 注意事項

### **プロジェクト名の制限**

- プロジェクト名は、同じアカウント内で一意である必要があります
- 既存のプロジェクト名と重複することはできません

### **既存プロジェクトを使用する場合**

- 既存の環境変数が上書きされる可能性があります
- 既存のデプロイ履歴が残ります

---

## 🎯 次のステップ

1. ✅ **既存プロジェクトを確認**
   - Vercelダッシュボードで確認

2. ✅ **既存プロジェクトを使用するか、新しい名前で作成**
   - 既存プロジェクトを使用: 方法1
   - 新しい名前で作成: 方法2

3. ✅ **環境変数を設定**
   - すべての環境変数が設定されているか確認

4. ✅ **デプロイ**
   - GitHub接続済み: 自動デプロイ
   - 未接続: `vercel --prod`
